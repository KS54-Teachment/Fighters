# DigitalSupport-Team-Fighters

## Цель проекта:
Обеспечить эффективную совместную работу ИТ-команды над документацией, инструкциями и базой знаний с использованием Git и GitHub. Подготовить структуру репозитория для итоговой защиты.
## Проблема:
 * Отсутствие единого workflow — нет чётких правил работы с ветками, коммитами и PR.
 * Конфликты слияния — возникают при одновременной правке одних файлов, требуют ручного разрешения.
 * Хаотичная структура репозитория — нет логичной иерархии папок, сложно найти нужные документы.
 * Неинформативные коммиты — сообщения не отражают сути изменений, затрудняют отслеживание истории.
 * Проблемы с правами доступа — участники не имеют нужных прав либо настройки слишком свободные, что ведёт к ошибкам.

## Вывод:
Для эффективной совместной работы ИТ‑команды над документацией в Git/GitHub нужно устранить ключевые проблемы: внедрить единый workflow с чёткими правилами работы с ветками, PR и коммитами, минимизировать конфликты слияния за счёт грамотного распределения задач и синхронизации веток, упорядочить структуру репозитория с логичной иерархией папок, установить стандарты для информативных сообщений коммитов и настроить оптимальные права доступа для участников — это обеспечит порядок в работе, снизит число ошибок и повысит продуктивность команды.
